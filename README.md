# VECTOR — CCNY

> *"Computer science is no more about computers than astronomy is about telescopes."* — Dijkstra

**VECTOR** is a student-led technical organization at The City College of New York. We build real software for NYC businesses, nonprofits, and community organizations — and teach students how to do it professionally.

This repo hosts the public-facing website for the organization.

---

## What is VECTOR

VECTOR is not a startup incubator. It's a **build environment**.

Students work in teams on scoped, client-facing projects — websites, dashboards, automations, AI tools — for real clients with real problems. The goal is to ship working software and develop the professional habits that coursework alone doesn't teach: scoping, client communication, delivery, and documentation.


---

## The Model

| For Students | For Clients |
|---|---|
| Gain experience beyond coursework | Get focused, affordable technical work |
| Work on real projects with real stakes | Transparent scope — no surprises |
| Build a portfolio with meaningful artifacts | Local team, dedicated point of contact |
| Learn client communication and delivery | Ongoing maintenance available |

**Who can join:** All CCNY students. CS and engineering students lead technical execution. Business, design, architecture, pre-med, and humanities students contribute to problem-finding, product thinking, and client relationships.

## Project Categories

**Web & Presence**
- Business websites (WordPress / Webflow)
- E-commerce stores (Shopify)
- Online booking systems
- Email marketing setup

**Data & AI**
- Business dashboards (Looker Studio / Python / Plotly)
- AI customer chatbots (OpenAI API)
- Internal admin tools (Retool / AppSmith / Flask + React)
- Automated reporting pipelines

**Ops & Automation**
- Inventory and sales automation (Airtable / Zapier / Make)
- Workflow digitization
- Document automation
- Monthly maintenance retainers

---

## How a Project Moves

```
01 Problem intake  →  02 Scope sprint  →  03 Build team  →  04 Validate  →  05 Artifact
```

Every project ends with a case study, demo, and documented GitHub repo — proof of work for both the student and the client.
